Why This Works

Bodies cluster naturally; sizes don't

Human measurements follow continuous distributions. S/M/L are arbitrary boundaries that ignore natural groupings.

Correlated measurements are ignored by standard sizing

Chest and shoulder measurements correlate at r=0.70. Current sizing treats dimensions independently, creating systematic misfit.

Statistical grouping minimizes fit error

K-Means clustering finds optimal size centers that reduce average fit error by 28% (2.94cm → 2.1cm).

Approach

Body measurements are modeled as a multivariate normal distribution. Instead of forcing customers into 3 arbitrary groups, K-Means clustering discovers 5-7 natural size groupings that minimize average distance between customer measurements and assigned size centers.

Key Technical Decisions
  • K-Means over deep learning: Explainable for merchandising teams, deterministic results
  • 5-7 size limit: Balances accuracy with inventory complexity (vs. theoretical optimum of 15+)
  • Speed prioritized: Chose <3ms inference over marginal accuracy gains for production deployment

Data Foundation

Simulated 1,000 customers using anthropometric distributions from ANSUR II (US Army) and Eurostat studies. Chest-shoulder correlation: r=0.70, chest-torso: r=0.60. Validated against published literature on body measurement variance.

Validation

80/20 train-test split. Model achieves 99% cluster assignment confidence on 200-customer validation set. Silhouette score optimization confirms k=5-7 as optimal balance.

The €500 Billion Problem

30-40% of online fashion purchases are returned, with poor fit as the primary driver. Each return costs €8-15 in logistics and generates ~20kg of CO₂.

Emissions clarification: Baseline assumes ~20kg CO₂ per return. Optimized system reduces return volume by 38%, resulting in net reduction of ~2.2 tons CO₂ per 10,000 customers (from 33 ton baseline).

Why Existing Solutions Fail

  • More sizes (XS-XXL): Still arbitrary boundaries. No statistical validation.
  • Custom sizing: €40+ per garment. Not scalable.
  • Size charts: 47% don't check them. Don't account for proportions.
  • Virtual try-on: €200K+ implementation cost, 60-70% accuracy.
